Documenting PHPRoad

The documentation of PHPRoad is completely built by and for the community of users. It covers things from installing the system and getting oriented with the system all the way to developing for the system. You can be among this awesome group of writers! There are many ways of contributing to these docs, even if you don't actually write yourself. You may:

  • Critique the writing.
  • Find missing information
  • Find incorrect information
  • Write new documents

Categorization

Our documents are broken into two main sections, user and developer docs. This allows us to talor the information being given to each type of reader. User docs cover using the system from a front-end view. Showing things such as how to install the system, update it, use the CMS and user manager. While the developer docs cover actually programming for PHPRoad. They disclose how to build custom modules for clients, ones that allow them to update without fear of their site breaking.

Submitting

Submitting any issues with the documentation can be done by submitting an issue or pull request on the docs repository. Please include where you see an issue and what you feel could make it better. The quickest way to get something updated, is to rewrite it and submit a pull request yourself. By doing so you save time for other writers and get the warm fuzzies of contributing. If you are unable to submit an issue on that repository, then you can contact others in the community and they can file it for you.
